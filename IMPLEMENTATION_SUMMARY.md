### 3. **Функционал старшего клинера**

#### Новые эндпоинты:
```python
# urls.py
path("<int:pk>/start-work/", views.order_start_work, name="order_start_work")
path("<int:pk>/finish-work/", views.order_finish_work, name="order_finish_work")
```

#### Логика работы:
1. **Начать работу** (`order_start_work`):
   - Сохраняет `work_started_at = timezone.now()`
   - Переводит статус в `IN_PROGRESS`
   - Проверка прав: только назначенный старший клинер или менеджер

2. **Завершить работу** (`order_finish_work`):
   - Сохраняет `work_finished_at = timezone.now()`
   - **Автоматически** переводит статус в `PENDING_REVIEW` (на проверку менеджеру)
   - Проверка прав: только назначенный старший клинер или менеджер

#### Интерфейс (`senior_cleaner.html`):
- ✅ Отображение статуса работы (не начата / в процессе / завершена)
- ✅ Список задач с исполнителями и статусами
- ✅ Информация о команде (назначенные клинеры)
- ✅ Кнопка "Начать работу" (если не начата)
- ✅ Кнопка "Завершить работу" (если в процессе)
- ✅ Ссылки на редактирование задач

## 🔄 Жизненный цикл заказа (обновлённый)

```
1. СОЗДАНИЕ (Оператор)

   status_operator = IN_PROGRESS
   
2. ПЕРЕДАЧА МЕНЕДЖЕРУ (Оператор)
   ↓
   status_operator = ACCEPTED
   status_manager = ASSIGNED
   
3. НАЗНАЧЕНИЕ КЛИНЕРОВ (Менеджер)
   ↓
   senior_cleaner = [выбран]
   cleaners = [выбраны]
   status_manager = IN_PROGRESS (автоматически)
   
4. НАЧАЛО РАБОТЫ (Старший клинер)
   ↓
   work_started_at = [текущее время]
   status_manager = IN_PROGRESS
   
5. ЗАВЕРШЕНИЕ РАБОТЫ (Старший клинер)
   ↓
   work_finished_at = [текущее время]
   status_manager = PENDING_REVIEW (автоматически)
   
6. ПРОВЕРКА КАЧЕСТВА (Менеджер)
   ↓
   quality_rating = [1-5]
   quality_comment = [текст]
   reviewed_at = [текущее время]
   reviewed_by = [менеджер]
   
   Решение:
   ├─ COMPLETED → Заказ завершён ✅
   └─ IN_PROGRESS → Вернуть на переделку 🔄
```

## 🚀 Как протестировать

### 1. Создание заказа (Оператор):
```
1. Войти как оператор
2. Создать заказ
3. Нажать "Передать менеджеру"
   → Статус оператора: ACCEPTED
   → Статус менеджера: ASSIGNED
```

### 2. Назначение клинеров (Менеджер):
```
1. Войти как менеджер
2. Открыть заказ со статусом ASSIGNED
3. Нажать "Редактировать заказ"
4. Выбрать старшего клинера
5. Выбрать обычных клинеров (Ctrl + клик)
6. Сохранить
   → Статус автоматически: IN_PROGRESS
```

### 3. Выполнение работы (Старший клинер):
```
1. Войти как старший клинер
2. Открыть назначенный заказ
3. Нажать "Начать работу"
   → work_started_at сохранён
4. Редактировать задачи (опционально)
5. Нажать "Завершить работу"
   → work_finished_at сохранён
   → Статус автоматически: PENDING_REVIEW
```

### 4. Проверка качества (Менеджер):
```
1. Войти как менеджер
2. Открыть заказ со статусом PENDING_REVIEW
3. Нажать "Проверить качество"
4. Выбрать оценку (1-5 звёзд)
5. Написать комментарий
6. Выбрать решение:
   - "Принять работу" → COMPLETED
   - "Требуется переделка" → IN_PROGRESS
7. Сохранить
   → Оценка и комментарий сохранены
   → Статус изменён согласно решению
```
